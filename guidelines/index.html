Guidelines

Whether you're new to the list, or just forgot the requirements, here are a list of rules for the GD Shotty Level List, involving submitting levels, records, and more.

Please follow these rules when submitting to the form, as these keep the list as enjoyable as possible.

Levels

  1. The level must be legitimately verified.

    2. To ensure all levels are possible, we ask that The level must be legitimately verified by you or another user.

  2. The level must created or verified by you.

    3. To prevent users from submitting levels without the creator's permission, we ask that your submission was created by you or verified by you.

    If the level is a copy of an already created level than the shotty version must be altered in some way AND must include credits to the original creator/creators.

  3. The level has to be possible on vanilla GD.

    4. ex needs to be possible without CBF.

  4. The level cannot exceed a difficulty of #600 on the ALEDL.

    5. If the level is found to exceed this difficulty it will be removed.

  5. The level must have basic structuring.

    6. While decorating isn't a requirement, levels must have basic structuring (ie cant fall out of the level/floating spikes) and cannot include mindless object spam.

    Starting 10/20/25, all Shotty levels must have the background set to a blue of some sort, not teal.

    Creators are not allowed to alter the camera controls of any level.

  6. The gameplay cannot be altered too drasticly.

    7. for example a ship section cannot be changed into a mini wave, but a mini wave can be turned into normal wave.

Records

  1. The record must be achieved without using hacks.

    2. for obvious reasons

  2. The record must be achieved on the level that is listed on the site.

    3. Please check the level ID before you submit a record.

  3. The record must clicks/taps in the video.

    4. Edited in clicks does not count, it must be audible (so no super quiet clicks), and videos with background noise is preferred.

  4. The recording must have a previous attempt and entire death animation shown before the completion.

    5. This does not apply if the completion is on the first attempt. Everyplay records are exempt from this rule.

  5. The record must not use secret routes or bug routes.

    6. Only use the intended way set by the creator. If you're unsure, check the verification video. Records with small mistakes (e.g. not hitting an orb) are permitted.

  6. The record must not an altered copy.

    7. Only a record of the unmodified level qualifies.
